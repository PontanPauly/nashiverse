# Nashiverse Star System Rewrite — Changes Summary

## Files Modified
- `src/components/constellation/Star.jsx` — 1386 → 1318 lines (rewritten)
- `src/lib/starConfig.js` — color palettes + glow decoupling

## What Changed

### 1. NMS-Style Luminous Cores (ALL 6 shaders)
**Before:** Core used `smoothstep` with `pow(0.8)` — this actually *expanded* the bright area (power < 1 brightens). Core radius was 0.08-0.12, overlapping heavily with the corona at 0.15. Everything blended into a homogeneous glow.

**After:** All shaders now use Gaussian falloffs (`exp(-d²*k)`) with layered intensity:
- **Pinpoint** (k=2500-4000): Tiny, blazing white center — overexposed HDR look
- **Hot core** (k=350-500): Tight bright region with warm white
- **Inner corona** (k=40-80): Where the star's primary color becomes visible
- **Outer corona**: Noise-varied atmospheric envelope
- **Atmosphere**: Very wide, faint, blending toward glow color

The math intentionally pushes intensity above 1.0 at center (pinpoint * 3.5, hotCore * 1.8), which gets naturally clamped by additive blending — creating the "overexposed" look real bright stars have.

### 2. Real Animation System (not just speed)
**Before:** `ANIMATION_TIME_SCALE` just multiplied delta time. "Twinkle" was the same animation running faster. All stars had a hardcoded `(0.88 + 0.12 * sin(time * 1.8))` shimmer baked into every shader.

**After:** Two independent systems:
- **Time scale** (pattern speed): How fast swirls/patterns evolve. Steady=0.15 (very slow), Dancing=1.5
- **Animation mode** (brightness character): JS-side `computeAnimBrightness()` modulates the brightness uniform per-frame:
  - **Steady (mode 0):** Rock solid 1.0
  - **Gentle (mode 1):** Subtle sinusoidal breathing ±8%
  - **Twinkle (mode 2):** Stochastic scintillation — 5 incommensurate sine frequencies create pseudo-random flicker, range ~0.55-1.1
  - **Breathing (mode 3):** Deep slow cycle ±15%
  - **Dancing (mode 4):** Dramatic pulses + pow(4) intensity bursts

Both StarSprite and OuterGlow animate independently — glow breathes with the same mode.

### 3. Atmospheric Effects Always Visible
**Before:** `AtmosphericHaze` and `DiffractionSpikes` only rendered on hover (`showAtmosphericEffects = isHovered || isFocused`). Idle stars were just StarSprite + OuterGlow — flat.

**After:**
- **AtmosphericHaze:** Always renders. Idle: `activeIntensity * 0.2`. Hover: `activeIntensity * 0.5`.
- **DiffractionSpikes:** Always renders for stars with brightness > 0.7. Idle: very subtle (`intensity * 0.15 * brightness`). Hover: prominent (`intensity * 0.5`).

### 4. GlowStyle Actually Works
**Before:** `GLOW_STYLES` defined `intensity`, `radius`, `falloff`, `pulsing`. Only `intensity` was ever read. "soft-halo" vs "sharp-rays" vs "flame" looked nearly identical.

**After:** Two new config maps translate glowStyle into shader parameters:
- `GLOW_FALLOFF_MAP`: Controls the exponential power in OuterGlow shader (soft-halo=2.0, sharp-rays=3.5, mist=1.5)
- `GLOW_RADIUS_MAP`: Controls the glow plane size (soft-halo=1.0x, mist=1.6x, sparkle=0.5x)

OuterGlow shader now accepts a `falloffPow` uniform that changes `exp(-pow(dist/r, falloffPow))` — higher values = sharper cutoff, lower = wider soft spread.

### 5. Glow Decoupled from Brightness
**Before:** `getStarVisuals()` computed `glow.intensity = glow.intensity * profile.brightness`. A dim star (brightness=0.3) had glow crushed to nearly invisible.

**After:** Formula changed to `glow.intensity * (0.4 + profile.brightness * 0.6)`. A star at brightness=0.3 now gets 58% glow instead of 30%. Even dim stars have visible atmosphere. Additionally, the Star component enforces a `dimFloor = 0.35` minimum.

### 6. Color Palette Contrast
**Before:** Many palettes had nearly identical primary/secondary (celestial: `#60A5FA` → `#3B82F6`, ~10% difference). Nebula and crystal shaders that mix between them showed no visible color transitions.

**After:** Secondary colors pushed much darker (celestial: `#60A5FA` → `#1E40AF`). Glow colors pushed lighter. This creates visible color gradients in corona regions and makes multi-color shaders (nebula, crystal, plasma) show actual color variety.

### 7. Dead Code Removed
- **DustCloud component** (90 lines): Defined but never rendered anywhere. Removed.
- **Hardcoded shimmer** in all shaders: `(0.88 + 0.12 * sin(time * 1.8))` removed — animation system handles this now.
- Removed duplicate code in animated vs non-animated render paths (consolidated pointer handlers).

### 8. Shader-Specific Improvements

| Shader | Key Changes |
|--------|------------|
| **Classic** | 5-layer Gaussian core (pinpoint → hotCore → innerCorona → outerCorona → atmosphere). Rays made subtler (0.12 vs 0.3 intensity). |
| **Nebula** | Stellar core (3 Gaussian layers) embedded above the nebula cloud. Core punches through the gas. |
| **Plasma** | Core (pinpoint + hotCore + corona) sits above swirling plasma. Electric arcs now orbit around a visible bright center. |
| **Crystal** | Gem core uses pinpoint + hotCore. Internal fire patterns now orbit around a bright center point, not a diffuse glow. |
| **Pulse** | Core breathes slightly with pulse frequency. Ripple rings emanate from a visible bright center. |
| **Nova** | Brightest core of all (pinpoint k=4000, multiplier 4.0). Explosion rays radiate from a blazing center. |
