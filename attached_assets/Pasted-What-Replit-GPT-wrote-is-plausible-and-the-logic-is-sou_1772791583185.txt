My take on the “Saving...” bug

Even if schema and API are correct now, the “Saving...” symptom almost always means one of these:

a thrown error not caught (loading never cleared)

an early return path that skips setLoading(false)

network request failing (400/500) and UI not surfacing it

backend response shape changed and UI is waiting for something that never arrives

That’s why I would force them to paste the Network response for one save attempt. It ends the guessing instantly.
